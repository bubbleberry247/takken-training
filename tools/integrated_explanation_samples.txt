## パターン分類

### パターンA: 詳細な選択肢別解説（AI分解可能）

#### サンプル1 (ID: 53625)
```
【正解は3です】
1→誤りです。RI計器による土の密度試験では、土の含水比のほかに乾燥密度や湿潤密度を求めることができ、盛土等の締固め度の判定等が行えますが、地盤の許容支持力を算定することはできません。地盤の許容支持力を求めるには、標準貫入試験を行い、N値を求めることにより算定できます。
2→誤りです。平板載荷試験では、地盤の反力係数を求めて支持力の算定をすることはできますが、地盤の厚さの確認はできません。地層の厚さを確認するには、ボーリング調査を実施してサンプルを採取する必要があります。
3→正しいです。問題文の通り、ポータブルコーン貫入試験では、地盤の貫入抵抗から地盤の硬さの目安のコーン指数を求めることにより、建設機械のトラフィカビリティー（建設機械の走行性）を判定できます。
4→誤りです。標準貫入試験では、N値を求めて、地盤の許容支持力を求めることができますが、盛土の締固め管理の判定はできません。盛土の締固め管理の判定は、RI計器による土の密度試験（最も一般的な方法です）により土の含水比を求めることで行うことができます。その他、含水比試験により水と土粒子の質量比を求めることにより判定することもできます。
```

**構造**:
- 冒頭: `【正解はXです】`
- 各選択肢: `N→判定。理由...`
- 区切り: スペース2個（目視では見えにくいが、実際は改行なしで連結）

#### サンプル2 (ID: 53626)

1→正しいです。土量の変化率は、実際の土工の結果から推定するのが最も的確な決め方です。
2→誤りです。土の掘削・運搬中の損失及び基礎地盤の沈下による盛土量の増加は、変化率には含まれていません。土量の変化率の算定は、土をほぐしたり締固めた際の体積比となります。
3→正しいです。土量の変化率Cは、地山の土量と締め固めた土量の体積比を測定して求めます。
4→正しいです。土量の変化率Lは、土工の運搬計画を立てる上で重要であり、土の密度が大きい場合には積載重量によって運搬量が定まります。

#### サンプル3 (ID: 53627)
```
【正解は1です】
1→誤りです。盛土の締固め管理技術は、品質規定方式（締固め度や支持力、飽和度等の品質を規定して管理を行う方式）を工法規定方式（締固め機械や転圧回数、まき出し厚さなどの工法を規定して管理を行う方式）にすることで、品質の均一化や過転圧の防止などに加え、締固め状況の早期把握による工程短縮がはかられます。

3→正しいです。まき出し厚さは、試験施工で決定したまき出し厚さと締固め回数による施工結果である締固め層厚分布の記録をもって、間接的に管理をすることができます。
4→正しいです。盛土の締固め管理では、締固め機械の走行位置を追尾・記録することで、規定の締固め度が得られる締固め回数の管理を厳密に行うことができます。
```

**構造**: サンプル1と同じ

#### サンプル5 (ID: 53629) - より複雑な例
```
正解は「固結工法は、セメントなどの固化材を土とかくはん混合し地盤を固結させることにより、変形の抑制、液状化防止などを目的とするもので、サンドコンパクションパイル工法などがある。」です。

選択肢1. 締固め工法は、地盤に砂などを圧入又は動的な荷重を与え地盤を締め固めることにより、液状化の防止や支持力増加をはかるなどを目的とするもので、振動棒工法などがある。
設問通りです。締固め工法は、地盤に砂などを圧入又は動的な荷重を与え締め固めたりする工法です。その目的は、液状化の防止、支持力増加です。 締固め工法は、【振動棒工法、バイブロフローテーション工法、サンドコンパクションパイル工法】などがあります。

選択肢2. 固結工法は、セメントなどの固化材を土とかくはん混合し地盤を固結させることにより、変形の抑制、液状化防止などを目的とするもので、サンドコンパクションパイル工法などがある。
誤りです。サンドコンパクションパイル工法は固結工法ではなく、締固め工法です。 固結工法には、【表層混合処理工法、深層混合処理工法、薬液注入工法】などがあります。

選択肢3. 荷重軽減工法は、軽量な材料による荷重軽減や地盤の挙動に対応しうる構造体をつくることにより、全沈下量の低減、安定性確保などを目的とするもので、カルバート工法などがある。
設問通りです。荷重軽減工法は、軽量な材料により荷重軽減を図り、地盤の挙動に対応する構造体をつくる工法です。 荷重軽減工法は、【カルバート工法、EPS工法、気泡混合軽量土工法】などがあります。

選択肢4. 圧密・排水工法は、地盤の排水や圧密促進によって地盤の強度を増加させることにより、道路供用後の残留沈下量の低減をはかるなどを目的とするもので、盛土載荷重工法などがある。
設問通りです。圧密・排水工法は、地盤の排水や圧密促進を図ることにより、地盤の強度を増加させる工法です。 圧密・排水工法は、【盛土荷重工法、緩速載荷工法、サンドマット工法】などがあります。
```

**構造**:
- 冒頭: 正解の選択肢全文
- 各選択肢: `選択肢N. 選択肢全文` + スペース多数 + 判定と理由

## AI分解の可能性評価

### 高精度で分解可能（パターンA詳細型）
- **対象**: サンプル1, 2, 3のような `N→判定。詳細理由...` 形式
- **信頼度**: 95%以上
- **条件**:
  - `1→`, `2→`, `3→`, `4→` または `選択肢1.`, `選択肢2.`, ... で明確に区切られている
  - 各選択肢に判定（正しい/誤り/適当/不適当）+ 理由が含まれている

### 中精度で分解可能（パターンA簡潔型）
- **対象**: サンプル8のような簡潔な判定+理由形式
- **信頼度**: 80%以上
- **条件**:
  - 選択肢区切りは明確
  - 理由が1-2文で短い

### 分解困難（パターンB）
- **対象**: サンプル6, 7, 9, 10のような統合解説のみ
- **信頼度**: 30%未満
- **問題点**:
  - 正解選択肢の説明しかない
  - 不正解選択肢の理由が書かれていない
  - AI推論で補完する必要があるが精度が低い

### 複雑形式（パターンA変形）
- **対象**: サンプル5のような選択肢全文引用型
- **信頼度**: 85%以上
- **条件**:
  - 選択肢全文が再掲されているため、マッチングは確実
  - 各選択肢の説明が詳細


## 分解戦略

### 戦略1: 正規表現ベース分解（パターンA向け）
```python
# パターン: "N→判定。理由..."
pattern = r'(?:^|[^\S\n]+)(\d+)[\.\→](.+?)(?=\d+[\.\→]|$)'

# パターン: "選択肢N. 文..."
pattern = r'選択肢(\d+)\.\s*(.+?)(?=選択肢\d+\.|$)'
```

### 戦略2: LLM支援分解（全パターン向け）
- **入力**: 問題文、選択肢4つ、統合解説
- **出力**: 各選択肢の説明文
- **プロンプト**: 「統合解説を選択肢ごとに分解してください」

### 戦略3: ハイブリッド
1. 正規表現で分解を試みる
2. 成功 → そのまま採用
3. 失敗 → LLM支援分解にフォールバック
4. LLMの出力を検証（選択肢数が4つか、判定ワードが含まれているか）


## 推奨アプローチ

**Phase 1**: 正規表現ベース分解スクリプトを実装
- パターンA詳細型（60-70%のケース）を高精度で処理
- 処理成功率を測定

**Phase 2**: LLM支援分解を追加
- Phase 1で処理できなかった残り30-40%をLLMで処理
- Claude/GPT-4を使用

**Phase 3**: 人手レビュー
- LLM出力の品質チェック
- 明らかな誤りを修正

**見積もり**:
- Phase 1: スクリプト開発 2時間、実行 10分
- Phase 2: LLM統合 2時間、実行 30分（API料金: 約$5-10）
- Phase 3: レビュー 4-6時間（192問 × 1.5-2分/問）
- **合計**: 8-10時間 + $5-10
