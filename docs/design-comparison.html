Claude design-system reference

sekisan-training の方向性を、4つのサンプルで比較

建築積算士の学習アプリとして「静かで信頼感がある」軸は共通にしつつ、どこまで業務寄りにするか、どこまで読み物寄りにするかの違いを見比べるための簡易モックです。

A. 無印良品風

静か / 素朴 / 余白重視
おすすめ度: 高

12週間で、静かに仕上げる。

余計な装飾を抑え、問題文と進捗だけを丁寧に見せる。積算ノートっぽさが最も出しやすい方向です。

生成色少なめ 紙っぽい配色 落ち着いた学習体験

ミニテスト

大きな余白と簡素なカードで、気持ちよく順番に進める。

年度別模試

年度の一覧も静かに見せて、主役は常に問題と学習リズム。

相性

図表・計算・解説を邪魔しない。

注意

地味すぎると達成感が弱くなる。

向いているなら: 落ち着いた受験アプリ、紙の教材の延長、和文中心。

B. SmartHR風

信頼感 / 業務UI / 明快
おすすめ度: 高

進捗管理が強い、整った学習ダッシュボード。

業務アプリの安心感があり、KPI・進捗・テスト管理をわかりやすく整理できます。

ニュートラル 視認性高い 実務感がある

今週のミニテスト

次にやることを明確に出しつつ、迷いを減らせる。

分野別ミニテスト

一覧や状態ラベルが見やすく、機能追加にも強い。

相性

学習管理機能と相性がよい。

注意

少し“社内システム感”が出やすい。

向いているなら: 進捗やデータの見やすさを最優先、今後も機能拡張したい。

C. Zenn風

読みやすい / 技術ノート / 軽い
おすすめ度: 中

解説が読みやすい、記事寄りの学習体験。

問題だけでなく解説をしっかり読む前提なら、軽い青アクセントと読みやすい余白が活きます。

解説重視 軽やか 記事っぽい

問題と解説

長めの解説や根拠の読みやすさを強くできる。

弱点復習

タグや補足テキストも軽く見せやすい。

相性

説明文が多い画面に向く。

注意

積算の“重厚感”は少し薄い。

向いているなら: 解説中心、読み物として気持ちよく使わせたい。

D. freee風

親しみ / 明快 / ややポップ
おすすめ度: 中

わかりやすく前向きな、軽快な学習UI。

進捗や開始ボタンを前向きに押しやすい雰囲気にできる一方、少し明るく軽めの印象になります。

親しみやすい CTA強め 導線が明快

学習スタート

開始ボタンや次アクションの勢いが最も出しやすい。

ダッシュボード

進捗カードやグラフも明るく整理できる。

相性

迷わず進める設計に強い。

注意

建築積算の渋さとは少し離れる。

向いているなら: 学習の継続性や押しやすさを優先したい。
