# UI/UX再設計案（性弱説ベース）

## 目的
- 試験勉強支援を「情報表示」ではなく「行動誘導」に変える
- 迷わせない最上段CTA（今日の一手）を常時提示する

## 画面構成（Home中心）
1. 今日の一手（最上段・1枚）
2. 進捗KPI（4枚）
3. 本日のテスト（カード）
4. 弱点から復習（タグ＋開始）
5. 週間推移（グラフ）
6. 最近の受験（表）
7. チーム進捗（上長のみ）

> Adminは別画面/別セクションに分離してHomeでは非表示

## UIワイヤー（テキスト）
```
[Header]  1級土木施工管理技士 14週間トレーニング

[今日の一手カード]
--------------------------------------------------

[開始]  [あとで]  [詳細]

--------------------------------------------------

--------------------------------------------------

[棒グラフ+線グラフ]

...

[チーム進捗]（上長のみ）

...
```

## 実装タスク分解（分担前提）

### バックエンド
1. 今日の一手判定ロジックの設計・実装
2. apiGetHome() に nextAction / reminders を追加
3. 未完了/期限/弱点/次回の優先順位ロジック
4. 例外設計（未ログイン/未設定/権限なし）

### フロント/UI
1. Homeレイアウト再配置（セクション順）
2. 今日の一手カードUIとCTA強調
3. 弱点復習のUI（1クリック導線）
4. KPIカードの視認性改善
5. テーブル/モバイル対応
