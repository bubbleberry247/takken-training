# Code Review Guidelines

## レビュー方針
- セキュリティ: ハードコードされた認証情報、SQLインジェクション、XSS を最優先で検出
- RPA固有: セレクタの脆弱性（動的ID依存）、タイムアウト設定の不足、エラーリカバリの欠如
- Python: PEP 8準拠、型ヒント、適切なエラーハンドリング
- 簡潔性: 過剰な抽象化やエラーハンドリングは指摘しない（動く最小単位が方針）

## 重要度レベル
- 🔴 Critical: セキュリティ脆弱性、データ損失リスク
- 🟡 Warning: パフォーマンス問題、保守性の低下
- 🔵 Suggestion: コード品質の改善提案
